## How to Contribute

1. Fork the repository and clone your fork to your local machine.
2. Create a new branch for your feature or bugfix:
   ```bash
   git checkout -b feature/your-feature
   ```

3.Make your changes, and ensure that all tests pass.

4.Commit your changes with a descriptive commit message:

```bash
git commit -m "Add feature X"
```

5. Push your branch to GitHub:

   ```bash
   git push origin feature/your-feature
   ```

6.Open a pull request on the main repository.

Code Guidelines
Use ESLint to ensure code consistency and avoid common errors.
Write unit tests for any new components or features.
Adhere to the existing component structure and naming conventions.
Reporting Bugs
If you encounter any issues or bugs, please open a GitHub issue and provide detailed steps to reproduce the problem.

Feature Requests
We welcome suggestions for new features! Open an issue with your idea, and we'll discuss whether it fits into the project.

Community and Support
Join our discussions on the GitHub Discussions page or reach out via GitHub issues if you need help.
